Ternary and Null Checking

                

                

					
Most Operators are Binary. Add adds two numbers...

					
While ternary operators operate on three

                    
echo ( $mightBeTrue ? "true" : "false" );

                    
$a = $mightBeTrue === true ? "true" : "false" ;

                    
$a = $mightBeTrue === null ? null : "Object is null" ;

                    
                        
$output = $obj == null ? "Object Can't String" : obj.ToString();

                    
                    
OK, So the "Ternary" operator is an if statement... 

                

				

                    
                    
Say we except the data

                    
                        
subscription: {    
    items: {
        data: [
            plan: {
                id: 'my_plan_id'
            }
        ]
    }
}
                    

                    
                    
We can get the value of the ID 

                    
                    
$value = $subscription->items->data->plan->id. 
                    

                    
                    
                

				

                    
Suppress errors using the exception stack..

                    
string $value;
try{ $subscription->items[0]->data->plan->id; }
catch (Exception e){ $value = "Object Incomplete";}
                    

                    
                    
But this is a unnecessary use of try catch used in a preventable logic flow

                

				

				
 Lets explicitly null check, because using exceptions for program flow is a bad idea.

				
string $value;
if ( $subscription != null \
    && $subscription->items != null 
    && $subscription->items[0] 
    && $subscription->items[0]->data != null 
    && $subscription->items[0]->data->plan != null 
    && $subscription->items[0]->data->plan->id != null){
    $value = $subscription->items->data[]->plan->id; 
}else {$value="Object Incomplete";

                    
But null checking is ugly

                

				

                    
 Using a ternary for a single check is usefull

                    
                        
$output = $obj == null ? "Object Incomplete" : $obj;

                    
                    
Chaining ternary operators a bad idea

                    
                        
$output = $subscription == null ? "Subscription Object Incomplete" 
    : $subscription->items == null ? "Items Incomplete" 
    : $subscription->items[0] == null ? "First Item Incomplete" 
    : $subscription->items[0]->plan == null ? "Plan Incomplete" 
    : $subscription->items[0]->plan->id == null ? "ID Incomplete" 
    : $subscription->items[0]->plan->id;

                    
                    
				

				

				
                    
Null Coalescing Operator

                    
Supresses Trying to get property of non-objet error

                    
                    
Null object returns null

                    
                        
$output = $obj ?? null
                        

                    
                    
                        
$output = $obj == null ? null : obj;
                        

                    
                    
Null object gets defaut value

                    
                        
$output = $obj ?? "Default Value"
                        

                    
                    
                        
$output = $obj == null ? "Default Value" : obj;
                        

                    
                

                
				

                    
Elvis Operator

                    
...behaves identical to the Null-Coalescing operator...

                    
When the variable itself is null.

                    

$a = null;
$out = $a ?? 'b'; // value is b
$out = $a ?: 'b'; // value is b 
$out = $c ?? 'b'; // value is b
$out = $c ?: 'b'; // Error: Undefined Variable $c

Spaceship Operator

                    
Spoiler: its used for sorting

                    
Because it is a 3 way comparitor

                

                

                    
<=>

                    
    
                        
  • -1 if left less than right
    • 
                        
  • 0 if left equal to right
    • 
                        
  • 1 if left greater than right
    • 
                    

                    
                    
$user_scores = [ 82 => [ 'score' => 1, 'time' => 6.442 ],
34 => [ 'score' => 1, 'time' => 5.646 ],
66 => [ 'score' => 3, 'time' => 1.554 ],
7 => [ 'score' => 2, 'time' => 4.442 ],
99 => [ 'score' => 4, 'time' => 3.646 ],
55 => [ 'score' => 1, 'time' => 2.554 ]
];

uasort($user_scores,function($a,$b){
    if($a['score'] == $b['score']){
        return $a['time'] <=> $b['time'];
    }
    return $b['score'] <=> $a['score'];
});
